# 專案計畫：GameForex (遊戲匯率通)

本專案旨在打造一個遊戲金融資訊導航站，彙整各類遊戲的幣值匯率、代儲價格及廠商評價，協助玩家在透明的資訊下做出交易決策。

---

## 1. 產品目標與定位 (Product Goal)

- **核心價值：** 用一致口徑比較遊戲幣價格，降低資訊落差與踩雷風險。
- **主要用戶：** 需要快速比價、重視風險提示、且會跨平台購買的玩家。
- **產品定位：** 資訊導航與風險輔助平台，不直接參與交易。

---

## 2. 核心功能規劃 (Core Features)

### 2.1 匯率追蹤系統 (Currency Tracker)
- **多來源數據整合：** 整合官方儲值比例、第三方平台均價、合作幣商報價。
- **統一換算基準：** 全部轉換為 `$1 TWD : 遊戲幣數量`，並標示資料來源與更新時間。
- **更新策略：** 採分級更新，低波動遊戲每日更新，高波動遊戲每 2 小時更新；若遇活動檔期可臨時提高頻率。
- **資料品質：** 顯示樣本數、異常值排除規則、資料新鮮度等級。

### 2.2 廠商與評價系統 (Vendor & Review)
- **廠商分類：** 官方管道、連鎖代儲、個體幣商。
- **信譽指標：** 外部評價 + 站內評分 + 履約歷史，形成可解釋分數。
- **風險提示：** 針對不同遊戲的政策風險提供等級標示（低/中/高）。
- **透明原則：** 公開評分權重與更新週期，提供申訴與更正流程。

### 2.3 導流與引導 (Referral System)
- **快速跳轉：** 商品頁深連結，減少使用者操作成本。
- **聯盟行銷：** 介面保留追蹤碼與來源標記。
- **揭露規範：** 所有商業合作連結明確標示，避免誤導。

## 3. MVP 與里程碑 (Delivery Plan)

### Phase 1 - MVP（4 週）
- 上線 2 至 3 款熱門遊戲。
- 每款遊戲至少 2 個資料來源。
- 完成基礎比價頁、來源標示、更新時間顯示。
- 支援 7 天趨勢圖（日級別）。
- 完成定義（DoD）：每款遊戲每日有效報價筆數達標、抓取成功率達標、法律檢核表完成。

### Phase 2 - Beta（8 週）
- 廠商頁與評分系統上線。
- 上線 30 天走勢圖與價格區間顯示。
- 增加風險提示模組與基礎告警。

### Phase 3 - Growth（12 週+）
- 活動/改版事件標記。
- 低價提醒與個人化追蹤。
- 擴充更多遊戲與更多來源合作。

## 4. 合規與資料治理 (Compliance & Data Governance)

### 4.1 資料來源合法性
- 官方售價通常無 API，且不易自動擷取，先由人工取得並紀錄。
- 人工維護欄位需包含維護人、最後驗證時間、佐證來源；超過 30 天未驗證即標示為待驗證。
- 第三方平台售價取得，優先順序：官方 API > 合作資料 feed > 合法可擷取頁面(後端 playwright 爬蟲)。
- 建立來源檢核表：是否允許抓取、存儲、展示、商用。
- 不符規範之來源不得上線。

### 4.2 內容責任與免責
- 明確聲明「本站僅供資訊參考，不保證即時成交價格」。
- 評分模型公開且可追溯，避免主觀黑箱。
- 提供廠商申訴與資料更正機制（含 SLA）。

### 4.3 安全與防濫用
- 站內評分機制加入頻率限制與異常偵測（單帳號/單裝置評分頻率上限、新帳號冷卻期、異常分數降權）。
- 保留關鍵操作稽核日誌。
- 個資最小化蒐集與保存期限管理。

## 5. 數據模型與品質指標 (Data Model & Quality)

### 5.1 核心欄位建議
- `game_id`, `source_id`, `quote_type`（official/market/vendor）
- `price_twd`, `currency_amount`, `unit_rate`
- `sample_size`, `collected_at`, `updated_at`, `confidence_score`

### 5.2 品質規則
- 異常值處理：採 IQR 或 MAD 規則排除極端值。
- 新鮮度目標（分級 SLA）：高波動遊戲 95% 報價在 2 小時內更新，低波動遊戲 95% 報價在 24 小時內更新。
- 來源健康度：追蹤抓取成功率、延遲、空值率。

## 6. 技術架構建議 (Tech Stack)

| 組件 | 推薦技術 | 說明 |
|---|---|---|
| 前端框架 | Vue 3.x (Composition API) | 響應式介面，適合高頻更新資料呈現。 |
| 前端語言 | TypeScript (Strict Mode) | 降低型別錯誤與維護成本。 |
| 狀態管理 | Pinia | 管理匯率快取、篩選條件與使用者偏好。 |
| 樣式系統 | Tailwind CSS + Naive UI | 快速建立一致 UI 與元件化設計。 |
| 後端開發 | Python (FastAPI) | 高效 API 與資料處理，易整合爬蟲。 |
| 排程與擷取 | Playwright + 排程器（APScheduler/Celery） | 應對動態頁面並支援重試與退避。 |
| 資料庫 | PostgreSQL | 儲存遊戲、報價、廠商、評價、歷史資料。 |
| 快取 | Redis | 提升查詢效能與降低後端壓力。 |
| 圖表呈現 | Lightweight Charts | 顯示 7/30 天趨勢與事件標記。 |

## 8. 成功指標 (KPI)

- **資料面：** 報價覆蓋率、更新新鮮度、錯價率。
- **產品面：** 比價頁轉化率、回訪率、平均停留時間。
- **商業面：** 導流點擊率、聯盟轉化率、每來源收益。

---

## 9. 主要風險與對策 (Risk Matrix)

- **來源封鎖風險：** 建立多來源備援與失敗降級策略；來源中斷時顯示最後可用價格與時間戳，並提示資料延遲。
- **資料偏差風險：** 公開口徑、顯示樣本數與信心分數。
- **法務風險：** 來源合法性審核與免責聲明上線前審查。
- **信任風險：** 評分透明化、業配標示、爭議處理機制。
